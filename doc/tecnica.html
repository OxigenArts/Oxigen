Introducción técnica

Este documento está hecho con la intención de brindar ayuda a los desarrolladores que deseen crear plugins, temas, extensiones o modificar el sistema base para adaptarlo a sus necesidades. Existen diversas clases que deben ser tratadas en el documento, con la intención de poder explicar su funcionamiento, así como también se mostrarán ejemplos de cómo crear extensiones, temas, etc.

Funcionamiento básico del sistema.

El sistema esta dividido en 6 carpetas principales:

  • /config: En esta carpeta se ubica el archivo de configuración de la base de datos, “database.php”. De igual manera, si se desea, se puede utilizar esta carpeta para colocar nuevos archivos de configuración requeridos por algún nuevo módulo o plugin para llevar un mejor orden.
  • /classes: En esta carpeta se colocan las clases nativas del sistema, así como también los plugins/módulos que se le deseen agregar. A pesar de que no es estrictamente necesario introducir los módulos en esta carpeta, se recomienda hacerlo.
  • /vistas: En esta carpeta estan los archivos de las vistas con extensión “.php” que serán mostrados según la petición realizada al servidor. Estos archivos son llamados por la clase Rutas(). Estas rutas por lo general se definen en el archivo “index.php” en la carpeta base del sistema, o en el archivo “rutas.php” de un tema.
  • /vistas/temas: Aquí se encuentran los archivos de los temas instalados.
  • /vistas/admin: Todos los archivos del dashboard se colocan aquí.
  • /media: Esta carpeta es utilizada para acceder y almacenar el contenido multimedia del sitio. Los archivos en general son guardados aquí y referenciados en la base de datos por la clase Media().

.htaccess

El archivo .htaccess es el archivo de configuración de Apache que permite, en este caso, redirigir todas las peticiones al archivo index. El archivo está nativamente configurando para funcionar junto a la clase Rutas() del sistema, ubicado en el archivo routes.php. 

RewriteEngine on
RewriteBase /directorio_instalacion
RewriteCond %{REQUEST_URI} !/directorio_instalacion/media/
RewriteCond %{REQUEST_FILENAME} !.*jpg$|.*gif$|.*png$|.*css$|.*js$|.*svg$|.*ttf$|.*woff$|.*woff2$|.*eot$
RewriteRule ^(.*)$ index.php?route=$1 [QSA,NC,L]

Este archivo puede ser modificado según la conveniencia del usuario o desarrollador.

index.php

Este archivo recibe y procesa todas las peticiones del servidor. Muestra la vista seleccionada por la ruta
